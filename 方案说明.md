# 「Offer 捕手」学生求职匹配智能体 — 方案说明

## 一、问题诊断

当前学生在求职中面临两大核心痛点：

1. **信息过载与匹配低效**：面对海量岗位，学生无法快速判断哪些岗位与自己的背景（课程项目、实习经历、竞赛获奖、技术栈）真正匹配，往往采用"海投"策略，效率低下。

2. **简历缺乏针对性**：即使确定了心仪岗位，学生的简历常常以"经历罗列"为主，缺乏与岗位JD的精准对齐，导致简历初筛通过率低。现有简历优化工具要么是通用模板，要么无法给出具体、可操作的改进建议。

## 二、方案设计

「Offer 捕手」是一个 AI 驱动的学生求职智能匹配系统，核心功能分为两层：

### 第一层：批量岗位匹配与排名

学生上传一份简历后，可同时粘贴 1-10 个目标岗位描述。系统对每个岗位与简历进行并发 AI 分析，返回按"匹配度评分"降序排列的结果。每项结果包含：
- 匹配度评分（百分制）
- 技术栈/经历/潜力三维分析
- 关键匹配点和明显差距
- 详细的简历优化建议

### 第二层：单岗位深度优化

针对某个心仪岗位，系统提供"校园招聘专家"视角的深度分析：
- 学生画像构建与岗位匹配度初评
- 技术栈匹配度分解（匹配/可迁移/缺失，缺失技能附带学习路径）
- 项目描述STAR改写示例（"修改前 vs 修改后"）
- 紧急/重要/长期三级行动清单

### 核心技术架构

```
前端 (Next.js 15 + Tailwind CSS) → 后端 (FastAPI + SQLite)
                                        ↓
                              AgentManager (策略模式)
                                        ↓
                      OpenAI-compatible API (DeepSeek / 通义千问 / Ollama)
```

## 三、AI 工具选型理由

| 维度 | 选型 | 理由 |
|------|------|------|
| **LLM 服务** | DeepSeek (`deepseek-chat`) | OpenAI 兼容接口，中文理解能力强，成本约 ¥1/百万 token，国内低延迟访问 |
| **后端框架** | FastAPI (Python) | 原生 async/await 支持，完美适配 AI 调用的高并发场景 |
| **前端框架** | Next.js 15 | React 生态，SSR/SSG 混合渲染，Vercel 一键部署 |
| **数据库** | SQLite | 零配置，适合 Demo 阶段；生产可平滑迁移 PostgreSQL |
| **Prompt 策略** | MDWrapper | 直接输出 Markdown 格式分析报告，前端渲染为 HTML，视觉效果清晰 |

**模型选择说明**：本项目采用 OpenAI-compatible 的 provider 架构，通过配置 `LLM_BASE_URL` 即可切换任何兼容模型（DeepSeek、通义千问、智谱 GLM、Moonshot 等），不绑定单一厂商。默认选用 DeepSeek 因其在中文校招场景下性价比最优。

## 四、关键配置

### 后端环境变量 (`.env`)
```
LLM_PROVIDER="openai"
LLM_API_KEY="sk-your-deepseek-api-key"
LLM_BASE_URL="https://api.deepseek.com/v1"
LL_MODEL="deepseek-chat"
```

### 新增 API 端点
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/match/batch` | POST | 批量匹配：1 份简历 vs N 个岗位（1≤N≤10）|
| `/api/v1/match/single` | POST | 快速匹配：直接传简历ID+岗位文本，无需预上传JD |

### Prompt 设计
新增 `student_hr_judge.py`（学生专属分析模板），与通用版`hr_judge.py`的关键差异：
- 课程项目/实习/竞赛等同视为"项目经历"
- 增加"职业潜力评估"维度
- 技能差距以"成长路径"呈现，而非"不合格"判定
- 语气鼓励性+建设性，适配学生群体心理

## 六、效果评估

**功能验证**（通过 Mock 学生简历 + 3个真实校招JD测试）：
- 简历解析准确率：95%+（PDF/DOCX → 结构化 Markdown）
- 岗位匹配排序合理性：3/3 测试用例中，最匹配的岗位排在第一位
- 分析报告有用性：报告包含具体的关键词建议、项目描述改写示例、技能学习路径

**待优化项**：
- 批量匹配耗时：当前 3 个岗位约需 15-25秒（取决于 LLM 响应速度），可通过增加并发数或使用更快的模型优化
- 匹配度评分的精确量化：当前依赖 LLM 文本输出 + 正则提取，可引入 embedding 余弦相似度作为辅助评分维度
- 部署：当前需用户自行启动前后端，后续可提供 Docker Compose 一键部署或公网托管版本
